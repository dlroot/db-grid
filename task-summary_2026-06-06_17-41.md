## Technical Details

### Server-Side DataSource Implementation
The `serverSideDatasource.getRows()` method:
1. Logs incoming request parameters (startRow, endRow, sortModel, filterModel)
2. Simulates API delay with `setTimeout(200-500ms)`
3. Generates requested rows on-the-fly using `generateServerRow()`
4. Applies sorting based on `sortModel` (first sort field only, for simplicity)
5. Applies filtering based on `filterModel` (case-insensitive contains match)
6. Calls `params.successCallback(rows, totalRowCount)`

### Memory Efficiency
Instead of generating all 100,000 rows at once, the implementation generates rows on-demand:
- Only generates rows for the requested range (`startRow` to `endRow`)
- Uses deterministic algorithm to generate row data based on ID
- This allows the demo to work with large datasets without memory issues

### Loading State
- `ServerSideService` emits `onLoadingChangedEvent` when loading state changes
- App component listens to this event and updates `serverLoading` signal
- UI shows "⏳ 加载中..." with pulse animation when loading
- UI shows "✅ 就绪" when not loading
